# 55-second demo script

1. **0–7 s — Problem.** “Most delay demos return a score. Flight Delay Risk turns schedule-only information into a calibrated review queue.”
2. **7–18 s — Single flight.** Enter airline, route, date and schedule; show probability, route support and local contributions.
3. **18–30 s — Batch ranking.** Upload the schedule and show the exact top-10% priority queue.
4. **30–40 s — Evidence.** Open validation: 250,000-row release sample, 50,453-flight untouched test, PR-AUC 0.2386 and Lift@10% 1.639×.
5. **40–48 s — Honesty.** Show drift and weekly confidence intervals; state that the final quarter is not reused for reselection.
6. **48–55 s — Engineering.** Show `/ready`, OpenAPI, Docker health checks and the GitHub architecture diagram.
